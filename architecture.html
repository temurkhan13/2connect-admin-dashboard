Matching Algorithm Flow

How the 7-step hybrid algorithm calculates matches for a newly onboarded user

Thresholds: Strong: ≥0.70 Worth Exploring: ≥0.55 Minimum: ≥0.50 Max matches/user: 50
Triggered by: onboarding.py:complete() → inline_matching_service.py:75
1
Get new user's embeddings SUPABASE
SELECT * FROM user_embeddings WHERE user_id = 'new_user_id'

Returns 6 vectors (each 1536 dimensions):
- industry_embedding, stage_embedding, goal_embedding
- geography_embedding, engagement_embedding, dealbreakers_embedding
2
Get ALL other users' embeddings SUPABASE
SELECT ue.*, u.allow_matching, u.is_active
FROM user_embeddings ue
JOIN users u ON ue.user_id = u.id
WHERE u.id != 'new_user_id'
AND u.allow_matching = true
AND u.is_active = true
AND u.onboarding_status = 'COMPLETED'
FOR EACH candidate user:
3
Calculate multi-vector base score ALGORITHM inline_matching_service.py:525-560
// Cosine similarity for each dimension
industry_sim = cosine_similarity(new_user.industry, candidate.industry)
stage_sim = cosine_similarity(new_user.stage, candidate.stage)
goal_sim = cosine_similarity(new_user.goal, candidate.goal)
geo_sim = cosine_similarity(new_user.geography, candidate.geography)
engagement_sim = cosine_similarity(new_user.engagement, candidate.engagement)
dealbreakers_sim = cosine_similarity(new_user.dealbreakers, candidate.dealbreakers)

// Weighted sum
BASE_SCORE = (industry_sim × 0.25) + (stage_sim × 0.20) + (goal_sim × 0.20)
+ (geo_sim × 0.15) + (engagement_sim × 0.10) + (dealbreakers_sim × 0.10)
4
Intent classification & complementarity ALGORITHM _get_intent_quality():769-816
Classify both users:
new_user.intent = "FOUNDER" (from primary_goal)
candidate.intent = "INVESTOR"
Look up intent quality:
INVESTOR ↔ FOUNDER = 1.0
MENTOR ↔ MENTEE = 1.0
COFOUNDER ↔ COFOUNDER = 0.8
SAME_TYPE = 0.3
5
Dealbreaker filters FILTER Line 651-667
IF new_user.intent == candidate.intent AND BLOCK_SAME_OBJECTIVE:
SKIP this candidate (two investors don't match)

IF explicit_dealbreaker_match(new_user, candidate):
SKIP this candidate
6
Bidirectional check ALGORITHM Line 669-686
// Does candidate also match well with new user?
forward_score = similarity(new_user → candidate)
reverse_score = similarity(candidate → new_user)

bidirectional_factor = sqrt(forward × reverse) / base_score
bidirectional_factor = clamp(0.5, bidirectional_factor, 1.5)
7
Activity & temporal boosts ALGORITHM Line 688-692
// Boost active users, decay inactive ones
activity_boost = candidate.login_count / max_logins // 0-1

days_since_active = (now - candidate.last_active).days
temporal_boost = max(0.85, 1.0 - (days_since_active × 0.005)) // 1.0 → 0.85 over 30 days
8
Calculate final score FORMULA Line 696-702
FINAL = BASE_SCORE × (0.7 + 0.3 × intent_quality) × bidirectional_factor × (0.9 + 0.1 × activity) × (0.95 + 0.05 × temporal)
Example: 0.72 × (0.7 + 0.3 × 1.0) × 1.1 × (0.9 + 0.1 × 0.8) × (0.95 + 0.05 × 0.95) = 0.78
END FOR EACH - Now have scores for all candidates
9
Filter and sort matches ALGORITHM
matches = [m for m in all_scores if m.score >= 0.50] // minimum threshold
matches.sort(by=score, descending=True)
matches = matches[:50] // max 50 per user

// Assign tiers
for match in matches:
if match.score >= 0.70: match.tier = "strong"
elif match.score >= 0.55: match.tier = "worth_exploring"
else: match.tier = "low"
Store
10

Feedback Learning Loop

How user feedback improves future matching accuracy

Embedding Adjustment: Base Rate: 2% per feedback Max Rate: 10% (prevents overfit) feedback_embedding_adjuster.py
Triggered when: User approves or declines a match
1
Feedback received BACKEND
POST /dashboard/decide-match
decision = "approved" | "declined"
reason_tags = ["wrong_industry"] // if declined
2
Backend proxies to AI Service AI SERVICE
POST /api/v1/feedback/match
Body: {
user_id: "user who gave feedback",
matched_user_id: "the other user",
decision: "approved" | "declined",
reason_tags: [...]
}
3
Get current embeddings SUPABASE
SELECT * FROM user_embeddings
WHERE user_id = feedback_giver_id

// Get all 6 embedding vectors
4
Calculate embedding adjustment ALGORITHM feedback_embedding_adjuster.py
IF decision == "approved":
// Move user's embedding CLOSER to matched user
adjustment_direction = matched_user.embedding - user.embedding
adjustment_rate = 0.02 // 2% base rate

ELIF decision == "declined":
// Move user's embedding AWAY from matched user
adjustment_direction = user.embedding - matched_user.embedding
adjustment_rate = 0.02

// Apply to relevant dimensions based on reason_tags
if "wrong_industry" in reason_tags:
user.industry_embedding += adjustment_direction * adjustment_rate
if "wrong_stage" in reason_tags:
user.stage_embedding += adjustment_direction * adjustment_rate
// ... etc for each dimension
5
Normalize adjusted embedding ALGORITHM
// Embeddings must be unit vectors for cosine similarity
adjusted_embedding = adjusted_embedding / ||adjusted_embedding||

// Verify: sum of squares = 1.0
6
Store updated embeddings SUPABASE
UPDATE user_embeddings
SET industry_embedding = '{...}'::vector,
stage_embedding = '{...}'::vector,
updated_at = NOW(),
feedback_adjustments_count = feedback_adjustments_count + 1
WHERE user_id = feedback_giver_id

Effect on Future Matches

If user APPROVED:
User's embedding moved 2% closer to approved match.
Future matches will score HIGHER for similar profiles.
"You liked SaaS investors, here are more SaaS investors"
If user DECLINED:
User's embedding moved 2% away from declined match.
Future matches will score LOWER for similar profiles.
"You passed on fintech, showing fewer fintech matches"
CONTINUOUS LEARNING
Each approve/decline adjusts embeddings by 2%
Over time, user's embedding space converges to their true preferences
Max adjustment capped at 10% to prevent overfit from outlier decisions
