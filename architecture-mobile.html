Authentication Flow

Login, registration, email verification, and token management

1
App Launch - Check Auth State SPLASH
Splash screen (index.tsx):
1. Check AsyncStorage for accessToken
2. If token exists → validate with /auth/me
3. If 401 → attempt token refresh
4. If refresh fails → clearAuth()
Routing decision:
No token → (auth) stack
Token + !onboarded → (onboarding)
Token + onboarded → (tabs)
2
Login / Signup (Single Screen) AUTH BUG-163 src/app/(auth)/login.tsx
Tab toggle (no navigation):
Login tab: email + password
Signup tab: name + email + pwd
Instant switch, no animation
API calls:
POST /auth/signin (login)
POST /auth/signup (register)
Google logo (BUG-161)
Response stored:
authStore.setUser(user)
authStore.setTokens(access, refresh)
3
Registration Flow AUTH src/app/(auth)/register.tsx
User input:
firstName, lastName, email, password
POST /auth/signup
Next step:
Navigate to verify-email.tsx
User enters OTP code from email
POST /auth/verify-email
4
Token Refresh (Axios Interceptor) AUTO src/services/api.ts
Request interceptor:
Inject Bearer token from authStore
headers.Authorization = `Bearer ${token}`
Response interceptor (401):
1. POST /auth/refresh
2. Update tokens in store
3. Retry original request
4. If fails → logout
OAuth Sign-in Options
Google Sign-In
POST /auth/google-signin
Body: { idToken: string }
Returns: { user, accessToken, refreshToken }
Apple Sign-In
POST /auth/apple-signin
Body: { identityToken, firstName?, lastName? }
Returns: { user, accessToken, refreshToken }

AI Onboarding Flow

Conversational AI extracts user profile slots through natural chat

1
Start Onboarding Session AI SERVICE
API call:
POST /onboarding/conversational/start
services/onboarding.ts:start()
Response:
{ sessionId, welcomeMessage, progress: 0 }
onboardingStore.startSession(sessionId)
2
Conversational Chat Loop CLAUDE SONNET 4.5 src/app/(onboarding)/chat.tsx
User sends message:
POST /conversational/chat
{ sessionId, message }
AI extracts slots:
MAX 3 slots per turn
primary_goal extracted FIRST
Objective-specific focus slots
Response:
{ aiResponse, extractedSlots, progress }
onboardingStore.updateSlots()
onboardingStore.setProgress()
3
Resume Upload (Optional Accelerator) OPTIONAL
Upload:
POST /onboarding/resume/upload
Content-Type: multipart/form-data
Supported: PDF, DOCX
Effect:
Resume text extracted (BUG-043)
Slots pre-extracted → Redis
Progress jumps significantly
4
Preview & Complete FINALIZE
Preview (80%+ progress):
Navigate to preview.tsx
Show extracted slots for review
User can edit before confirming
Complete:
POST /conversational/complete
Backend: persona → embeddings → summary
authStore.updateOnboardingComplete(true)
Navigate to (tabs) dashboard
onboardingStore State (Zustand + AsyncStorage)
{
  sessionId: string | null,
  isActive: boolean,
  progressPercentage: number // 0-100,
  messages: ChatMessage[],
  isTyping: boolean,
  extractedSlots: { primary_goal, industry_focus, stage_preference, ... },
  resumeUploaded: boolean,
  resumeFileName: string
}
