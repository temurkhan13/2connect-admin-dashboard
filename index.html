    AI Pipeline Architecture

    How onboarding → matching → feedback learning works

    1

    User Onboarding

    CONVERSATIONAL CHAT

    User answers questions naturally via chat. LLM (Claude Haiku) extracts structured data:

    primary_goal "Seeking Investment", "Looking to Invest", etc.
    requirements What user NEEDS (free text)
    offerings What user CAN GIVE (free text)
    industry Fintech, SaaS, AI/ML, etc.
    stage Pre-seed, Seed, Series A, etc.
    geography UK, US, Europe, Global, etc.

    ⚡ Slots are ADAPTIVE

    Different questions based on primary_goal:

    Seeking Investment → funding_need, company_stage, team_size
    Looking to Invest → check_size, investment_thesis, portfolio
    Finding Co-founder → skills_have, skills_need, commitment
    Hiring Talent → role_type, seniority, compensation
    Seeking Mentor → mentorship_areas, time_commitment

    Hybrid Matching

    INLINE (Event-Based)

    Runs IMMEDIATELY when user completes onboarding. Not a cron job.

    // Final score formula:

    FINAL = BASE_SCORE

    × (0.7 + 0.3 × intent_quality)

    × bidirectional_factor

    × (0.9 + 0.1 × activity_boost)

    × (0.95 + 0.05 × temporal_boost)

    // If FINAL ≥ 0.7 → CREATE MATCH

    5

    Feedback Learning

    ONGOING

    User feedback (👍/👎) adjusts embeddings:

    Base rate: 2% adjustment per feedback
    Max rate: 10% (prevents overfit)
    Normalization: Vectors normalized to prevent inflation

    Example:

    User gives 👍 on match

    → User's requirements embedding

    → Slightly adjusted toward match's offerings

    → Future matches improve

    Hybrid Matching Algorithm (7 Steps)

    STEP 1 Multi-Vector Base Score

    industry_similarity × 0.25

    + stage_similarity × 0.20

    + primary_goal_similarity × 0.20

    + geography_similarity × 0.15

    + engagement_similarity × 0.10

    + dealbreakers_similarity × 0.10

    = BASE_SCORE (0.0 - 1.0)

    STEP 2 Intent Classification

    Analyzes persona text for keywords to determine intent:

    INVESTOR_FOUNDER

    "investor", "fund", "portfolio"

    FOUNDER_INVESTOR

    "raising", "seeking investment"

    MENTOR_MENTEE

    "mentor", "guide", "advise"

    MENTEE_MENTOR

    "learn from", "need advice"

    COFOUNDER

    "co-founder", "founding team"

    TALENT_SEEKING

    "hiring", "recruit"

    STEP 3 Intent Complementarity

    Perfect (1.0)

    INVESTOR ↔ FOUNDER

    MENTOR ↔ MENTEE

    TALENT_SEEKING ↔ OPPORTUNITY

    Good (0.8)

    COFOUNDER ↔ COFOUNDER

    PARTNERSHIP ↔ PARTNERSHIP

    Poor (0.3)

    Same-seeking intents

    INVESTOR ↔ INVESTOR

    STEP 4 Filters (Remove Bad Matches)

    BLOCK_SAME_OBJECTIVE

    Skip investor↔investor, founder↔founder matches

    ENFORCE_HARD_DEALBREAKERS

    Skip if user's dealbreakers triggered

    STEP 5 Bidirectional Scoring

    forward = My requirements → Their offerings

    reverse = My offerings → Their requirements

    bidirectional_factor = √(forward × reverse) / base

    (Both parties must benefit - geometric mean)

    STEP 6 Boost Factors

    activity_boost (0-1)

    Recently active users ranked higher

    temporal_boost (0-1)

    New users get 14-day visibility boost

    STEP 7 Final Score & Threshold

    FINAL = BASE × (0.7 + 0.3×intent) × bidirectional × (0.9 + 0.1×activity) × (0.95 + 0.05×temporal)

    If FINAL ≥ 0.7 (threshold) → ✅ CREATE MATCH

    Adaptive Slot Groups

    Slots change based on user's primary_goal:

    CORE_SLOTS (Always)

    primary_goal, requirements, offerings, user_type, industry_focus, stage_preference, geography

    INVESTOR_SLOTS

    check_size, portfolio_size, investment_thesis

    When: "Looking to Invest"

    FOUNDER_SLOTS

    company_stage, funding_need, team_size

    When: "Seeking Investment"

    COFOUNDER_SLOTS

    skills_have, skills_need, commitment_level, equity_expectations

    When: "Finding Co-founder"

    HIRING_SLOTS

    role_type, seniority_level, remote_preference, compensation_range

    When: "Hiring Talent"

    MENTORSHIP_SLOTS

    mentorship_areas, experience_level, time_commitment

    When: "Seeking Mentor" or "Offering Advisory"

    Confidence Scoring

    Each extracted value gets a confidence score (0.0-1.0):

    Exact text match

    User says "Seed" for stage

    0.95

    Multi-word match

    "early stage startup" → "Seed"

    0.75

    Single keyword match

    "early" → "Pre-seed or Seed"

    0.70

    Dual-role detected

    "I'm a founder who also invests"

    0.45

    Number without units

    "I need 500" (500 what?)

    0.35

    ⚡ Low confidence triggers clarification

    If confidence < 0.7, system asks follow-up question to clarify
