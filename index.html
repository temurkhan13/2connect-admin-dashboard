    AI Pipeline Architecture

    How onboarding → matching → feedback learning works

    1

    User Onboarding

    CONVERSATIONAL CHAT

    User answers questions naturally via chat. LLM (Claude Sonnet 4.5) extracts structured data:

    primary_goal "Seeking Investment", "Looking to Invest", etc.
    requirements What user NEEDS (free text)
    offerings What user CAN GIVE (free text)
    industry Fintech, SaaS, AI/ML, etc.
    stage Pre-seed, Seed, Series A, etc.
    geography UK, US, Europe, Global, etc.

    ⚡ Slots are ADAPTIVE

    Different questions based on primary_goal:

    Seeking Investment → funding_need, company_stage, team_size
    Looking to Invest → check_size, investment_thesis, portfolio
    Finding Co-founder → skills_have, skills_need, commitment
    Hiring Talent → role_type, seniority, compensation
    Seeking Mentor → mentorship_areas, time_commitment
    Looking for Job → job_title, experience_years, salary_range
    Seeking Partnership → partnership_type, collaboration_scope

    Feedback Learning

    ONGOING

    User feedback (👍/👎) adjusts embeddings:

    Base rate: 2% adjustment per feedback
    Max rate: 10% (prevents overfit)
    Normalization: Vectors normalized to prevent inflation

    Example:

    User gives 👍 on match

    → User's requirements embedding

    → Slightly adjusted toward match's offerings

    → Future matches improve

    Hybrid Matching Algorithm (8 Steps)

    Updated March 28, 2026 — Additive weighted formula with tiered sorting. Validated across 64 users, 1,270 match pairs, zero wrong matches.

    STEP 1 Bidirectional Core Score

    forward = cosine(my_requirements_embedding, their_offerings_embedding)

    reverse = cosine(my_offerings_embedding, their_requirements_embedding)

    core_score = sqrt(forward × reverse)

    Geometric mean ensures both parties benefit. Asymmetry penalty if |fwd-rev| > 0.15.

    STEP 2 Dimensional Alignment Score

    industry_combined × 3.0

    + stage_combined × 2.0

    + skills_combined × 1.0

    + culture_combined × 0.5

    = dimension_score (weighted average of cosine similarities)

    STEP 3 Intent Classification

    Priority 1: primary_goal slot (confidence 0.95). Priority 2: keyword scoring across persona text.

    INVESTOR_FOUNDER

    "invest", "fund", "portfolio"

    FOUNDER_INVESTOR

    "raise funding", "seeking capital"

    TALENT_SEEKING

    "hire talent", "recruit"

    OPPORTUNITY_SEEKING

    "find new job", "career"

    SERVICE_PROVIDER

    "offer services", "consulting"

    PARTNERSHIP

    "explore partnerships", "B2B"

    COFOUNDER

    "find co-founder"

    MENTOR / MENTEE

    "mentor", "seek mentorship"

    RECRUITER

    "recruiter", "talent acquisition"

    GENERAL

    Fallback: "general networking"

    STEP 4 Intent Complementarity (40+ pairs)

    Perfect (1.0)

    INVESTOR ↔ FOUNDER

    MENTOR ↔ MENTEE

    TALENT ↔ OPPORTUNITY

    RECRUITER ↔ TALENT (0.95)

    Strong (0.80-0.90)

    COFOUNDER ↔ COFOUNDER (0.90)

    SERVICE ↔ FOUNDER (0.85)

    PARTNERSHIP ↔ PARTNERSHIP (0.85)

    PARTNERSHIP ↔ SERVICE (0.80)

    Moderate (0.50-0.70)

    SERVICE ↔ TALENT (0.70)

    PARTNERSHIP ↔ FOUNDER (0.65)

    PARTNERSHIP ↔ INVESTOR (0.60)

    SERVICE ↔ SERVICE (0.50)

    Blocked

    INVESTOR ↔ INVESTOR

    FOUNDER ↔ FOUNDER

    INVESTOR ↔ SERVICE_PROVIDER

    INVESTOR ↔ PARTNERSHIP

    INVESTOR ↔ TALENT_SEEKING

    INVESTOR ↔ GENERAL

    MENTEE ↔ OPPORTUNITY

    STEP 5 Hard Filters

    BLOCK_SAME_OBJECTIVE

    Two investors, two founders raising, two job seekers, two hiring companies

    BLOCK_INVESTOR_NON_FOUNDER

    Investors only see founders. Service providers, partnerships, general blocked from investor lists.

    ENFORCE_DEALBREAKERS

    Skip if user's hard dealbreakers triggered by candidate profile

    STEP 6 Final Score Formula (Additive Weighted)

    // Phase 1: Base total

    base = core_score × 0.40 + dimension_score × 0.35 + signal_score × 0.10

    // Phase 2: Intent as hard multiplier

    if intent_quality < 0.50: final = base × (IQ/0.50) + IQ × 0.15 // Crush bad pairs

    if intent_quality >= 0.95: final = base × 1.15 + IQ × 0.15 // Boost perfect pairs

    else: final = base + IQ × 0.15 // Standard additive

    // Phase 3: Spread + ceiling

    final = final ^ 0.85 // Power scaling for spread

    final = min(final, intent_quality) // Score NEVER exceeds intent quality

    STEP 7 Tiered Ranking

    For transactional intents (founder, investor, talent, opportunity):

    1. Perfect complementary matches (IQ >= 0.95) ALWAYS rank first

    2. Within each tier, ranked by score descending

    Result: Investors always appear above service providers/partners for founders, regardless of embedding similarity.

    For non-transactional intents (partnership, service_provider, cofounder): ranked purely by score.

    STEP 8 Threshold & Storage

    If score >= 0.50 (threshold) → Store match

    Score tiers: >= 75% STRONG | >= 60% WORTH_EXPLORING | < 60% LOW

    Max 50 matches per user. Synced to backend + match cache in Supabase.

    Confidence Scoring

    Each extracted value gets a confidence score (0.0-1.0):

    Exact text match

    User says "Seed" for stage

    0.95

    Multi-word match

    "early stage startup" → "Seed"

    0.75

    Single keyword match

    "early" → "Pre-seed or Seed"

    0.70

    Dual-role detected

    "I'm a founder who also invests"

    0.45

    Number without units

    "I need 500" (500 what?)

    0.35

    ⚡ Low confidence triggers clarification

    If confidence < 0.7, system asks follow-up question to clarify
